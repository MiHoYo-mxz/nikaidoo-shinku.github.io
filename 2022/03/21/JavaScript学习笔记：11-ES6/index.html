# ES6 简介

  • 什么是 ES6：ES 的全称是 ECMAScript , 它是由 ECMA 国际标准化组织，制定的一项脚本语言的标准化规范。es6 的全称是：ECMAScript6.0

  • ECMAScript 6.0（以下简称 ES6）是 JavaScript 语言的下一代标准，于 2015 年 6 月正式发布。目标 “使得 JavaScript 语言可以用来编写复杂的大型应用程序，成为企业级开发语言”

  • ES6 实际上是一个泛指，泛指 ES2015 及后续的版本。

  • ECMAScript 6（以下简称 ES6）是 JavaScript 语言的下一代标准。因为当前版本的 ES6 是在 2015 年发布的，所以又称 ECMAScript 2015。也就是说，ES6 就是 ES2015。

  • ECMAScript 是一种由 Ecma 国际（前身为欧洲计算机制造商协会，European Computer Manufacturers Association）通过 ECMA-262 标准化的脚本程序设计语言。这种语言在万维网上应用广泛，它往往被称为 JavaScript 或 JScript，所以它可以理解为是 JavaScript 的一个标准，但实际上后两者是 ECMA-262 标准的实现和扩展。

  • ES6 + 提供了许多新的语法和编程特性以提高 ECMAScript 的开发效率并优化 ECMAScript 的开发体验。ECMAScript 的发展速度在不断加快，影响范围越来越大，除了 Web 前端开发以外，借助着 Node.js 的力量在服务器、桌面端甚至硬件设备等领域中也发光发热着。

  • 学习参考: https://es6.ruanyifeng.com/

    # ECMAScript 和 js 的关系

    • 1996 年 11 月，JavaScript 的创造者网景公司，决定将 JavaScript 提交给国际标准化组织 ECMA，希望这种语言能够成为国际标准。
    • 1997 年，ECMA 发布 262 号标准文件（ECMA-262）的第一版，规定了浏览器脚本语言的标准，并将这种语言称为 ECMAScript，这个版本就是 1.0 版。
    • ECMAScript 和 JavaScript 的关系是，ECMAScript 是一种规范，JavaScript 是 ECMAScript 的一种实现（还有 Jscript、ActionScript、Node）。日常场合，这两个词是可以互换的。

# 块级作用域 block

  • 在 es5 中的是没有块级作用域，当时我们只能通过 function 来划分作用域。在 es6 中实现了块级作用域，具体来说，就是通过 let 关键字实现的。
  • 块级作用域是通过一对大括号 (花括号) 划分的 另外还需要结合 let 与 const 才能实现块级作用域

# let 关键字

  • ES6 中新增的用于声明变量的关键字。 与 var 类似，用来定义变量。

  • ❤let 定义的变量特点：

    1. 块级作用域 (第三种作用域)。let 所声明的变量，只在 let 命令所在的代码块内有效。

    2. let 没有 "变量提升" 现象。

    3. 不会成为 window 对象的属性

    4. 暂时性死区（TDZ）

      • ES6 明确规定，如果区块 { } 中存在 let 和 const 命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域。凡是在声明之前就使用这些变量，就会报错。
      • 在代码块内，使用 let 命令声明变量之前，该变量都是不可用的。这在语法上，称为 “暂时性死区”（temporal dead zone，简称 TDZ）
      • 说白了，就是要先定义变量，再使用。

    5. 不能重复定义 (不能重复声明)

    # let 与 var 的区别

    • 块级作用域变量获取，在循环中块级作用域会针对每一次迭代 (循环) 创建一个新的作用域

    • for 循环还有一个特别之处，就是设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域。

    # Object.freeze () 方法

    • Object.freeze () 方法可以冻结一个对象。一个被冻结的对象再也不能被修改；冻结了一个对象则不能向这个对象添加新的属性，不能删除已有属性，不能修改该对象已有属性的可枚举性、可配置性、可写性，以及不能修改已有属性的值。此外，冻结一个对象后该对象的原型也不能被修改。

# 解构赋值 (重点)- 数组解构

# 什么是解构赋值

  • ES6 允许按照一定模式，从 "数组" 和 "对象" 中提取值，按照对应位置，对变量进行一一赋值，这被称为解构（Destructuring）。

  • 作用：是对变量进行赋值。

  • 变量的值的来源：是数组或者对象

  • 规则：有一定的模式

# 解构赋值 (重点)- 对象解构

  • 本质：按属性名的一一对应关系来进行赋值

    # 解构规则

    • let {属性名 1: 变量名 1, 属性名 2: 变量名 2, 属性名 3: 变量名 3..} = 对象；
    • 如果属性名和变量名一样的时候，可以省略成如下: let {变量名 1, 变量名 2, 变量名 3..} = 对象；
    • 必须要通过键名进行关联。
    • 没有键名就是 undefined
    • 对象的解构赋值的内部机制，是先找到同名属性，然后再赋给对应的变量

# 解构赋值的相关应用

  • 应用 1 - 使用数组解构，交换两个变量的值

  • 应用 2 - 对象的解构赋值，可以很方便地将现有对象的属性和方法，赋值到某个变量。

  • 应用 3 - 函数中形式参数中的解构赋值

  • 应用 4 - 快速获取从函数中返回的多个值

  • 也可以对 ajax 请求到的数据进行解构

# 简化柯里化函数

  • 什么是柯里化函数，为什么要柯里化函数？

    1. 函数柯里化（currying）又称部分求值。一个 currying 的函数首先会接受一些参数，接受了这些参数之后，该函数并不会立即求值，而是继续返回另外一个函数，刚才传入的参数在函数形成的闭包中被保存起来。待到函数被真正需要求值的时候，之前传入的所有参数都会被一次性用于求值。

    2. 函数柯里化的重要意义在于可以把函数完全变成「接受一个参数；返回一个值」的固定形式，这样对于讨论和优化会更加方便。

    3. 柯里化在做函数组合时非常有用，让人们将关注的重点聚焦到函数本身，而不因冗余的数据参数分散注意力。

    4. 函数柯里化是一种函数式编程的思想，用已有的函数组合成新的函数，而柯里化函数每消费一个参数，都会返回一个新的部分配置的函数，这位函数组合提供了更灵活的手段，使得接口更为流畅。

  • 把柯里化函数改成箭头函数

# 箭头函数的特点 重点

  • 特点：

    • 箭头函数不能作为构造函数配合 new 关键字使用。箭头函数本身就没有实例 (this)。
    • 箭头函数不可以使用 arguments 对象，该对象在函数体内不存在。
    • 箭头函数根本没有自己的 this , 箭头函数内部的 this 指向声明这段代码的作用域。 ❤
    • 箭头函数不能作为 Generator 函数。

# 箭头函数优缺点

# 优点：

  • 箭头函数有更加简洁的语法，可以进行隐式返回。
  • 箭头函数不绑定 this ，会捕获其所在上下文的 this ，作为自己的 this ，解决了 this 的指向问题。
  • 箭头函数不绑定 arguments ，可以使用 ...args 代替。

# 缺点：

  • 箭头函数不能用作构造函数，不能通过 new 关键字创建实例。
  • 箭头函数不能用作对象的方法，因为它们没有自己的 this 。
  • 箭头函数不能访问 arguments 对象，只能用剩余参数来获取参数列表。

# 参数默认值

  • 参数默认值 在形参的后面使用 = 号赋值一个默认值 没有接收到对应的实参的时候，就会使用默认值

  • 细节：

    1. 参数变量是默认声明的，所以不能用 let 或 const 再次声明
    2. 定义了默认值的参数，应该是函数的尾参数 (尾参数指的就是在形参列表的最后)
    3. 传递 null 并不会使得形参使用默认值，参数的值将会变成 null
    4. 传递 undefined 会使得形参使用默认值
    5. 通过函数名.length 属性可以获取形参的总个数，但是会忽略设置了默认值的形参
    6. 函数的任何一个形参一旦被设置默认值，函数在声明初始化时。所有的参数都将变为独立作用域的参数 (相同形参名字的时候，如果有参数设置默认值，将报错)
    7. 函数的默认值是惰性求值的，默认值只会在函数调用的时候才会执行

# new Set () 的优点：

  • 可以实现数组或字符串的去重
  • 可以实现集合的并集、交集、差集等运算

  • 没有提供过滤、映射等方法，需要自己实现

# Proxy 对象优势或优点

  • Proxy 对象有很多优势或优点，我觉得主要有以下几点：

    • Proxy 对象可以实现对目标对象的任意操作的拦截和自定义，提供了一种灵活和强大的方式来修改或扩展对象的行为。

    • Proxy 对象可以与 Reflect 对象结合使用，以实现更简洁和一致的代理功能，同时保留目标对象的默认行为。

    • Proxy 对象可以用于实现一些高级的功能，如数据绑定、验证、格式化、私有属性、虚拟化等，提高了代码的可读性和可维护性。

# Proxy 对象局限性或缺点

  • Proxy 对象的局限性或缺点如下：

    • Proxy 对象无法代理目标对象的内部属性，如 [[Prototype]]、[[Extensible]] 等。

    • Proxy 对象无法监视目标对象的私有属性，因为它们不可枚举。

    • Proxy 对象无法拦截目标对象的静态方法，如 Object.keys ()、Object.assign () 等。

    • Proxy 对象无法完全模拟原生对象，如数组、函数、日期等，因为它们有一些内部行为和特性，如 length 属性、name 属性、toString () 方法等。

    • Proxy 对象可能会影响目标对象的性能和内存占用，因为它需要额外的操作和存储空间。

# symbol 数据类型

  • es6 新增数据类型 symbol

  • ES5 的对象属性名都是字符串，这容易造成属性名的冲突。比如，你使用了一个他人提供的对象，但又想为这个对象添加新的方法（mixin 模式），新方法的名字就有可能与现有方法产生冲突。如果有一种机制，保证每个属性的名字都是独一无二的就好了，这样就从根本上防止属性名的冲突。这就是 ES6 引入 Symbol 的原因。

  • ES6 引入了一种新的原始数据类型 Symbol，表示独一无二的值。它属于 JavaScript 语言的原生数据类型之一，其他数据类型是：undefined、null、布尔值（Boolean）、字符串（String）、数值（Number）、大整数（BigInt）、对象（Object）。
