# Promise

  • Promise 是一个对象，它代表了一个异步操作的最终 完成 或者 失败。

  • 本质上 Promise 是一个函数返回的对象，我们可以在它上面绑定回调函数，这样我们就不需要在一开始把回调函数作为参数传入这个函数了。

  • Promise 是异步编程的一种解决方案，比传统的解决方案 —— 回调函数和事件 —— 更合理和更强大。它由社区最早提出和实现，ES6 将其写进了语言标准，统一了用法，原生提供了 Promise 对象。

  • 所谓 Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。从语法上说，Promise 是一个对象，从它可以获取异步操作的消息。Promise 提供统一的 API，各种异步操作都可以用同样的方法进行处理。

    # Promise 对象特点：

    1. "对象的状态不受外界影响"。Promise 对象代表一个异步操作，有三种状态：pending（进行中，正在等待结果）、fulfilled（已成功）和 rejected（已失败）。只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。这也是 Promise 这个名字的由来，它的英语意思就是 “承诺”，表示其他手段无法改变。
    2. "一旦状态改变，就不会再变"，任何时候都可以得到这个结果。Promise 对象的状态改变，只有两种可能：从 pending 变为 fulfilled 和从 pending 变为 rejected。只要这两种情况发生，状态就凝固了，不会再变了，会一直保持这个结果，这时就称为 resolved（已定型）。如果改变已经发生了，你再对 Promise 对象添加回调函数，也会立即得到这个结果。这与事件（Event）完全不同，事件的特点是，如果你错过了它，再去监听，是得不到结果的。

# 回调地狱

  • 回调函数的相关嵌套，就是回调地狱 回调函数里面嵌套回调函数，再嵌套回调函数

  • Promise 的出现主要是解决回调地狱的问题，比如你需要结果需要请求很多个接口，这些接口的参数需要另外那个的接口返回的数据作为依赖，这样就需要我们一层嵌套一层，但是有了 Promise 我们就无需嵌套

# promise A + 规范 要记

  • 由于 promise 本身出自于民间，就有很多不同的实现版本。

  • ES6 将 promise 纳入自己规范的时候，也遵循了一个相应的标准 —— Promise A + 规范。

  • 将其归纳为 4321 规范。

    • 4：4 大术语 解决（fulfill）终值（eventual value） 拒绝（reject） 据因（reason）
    • 3：3 种状态 等待状态（进行中，正在等待结果）pending 成功状态（已成功，解决）fulfilled 失败状态（已失败，拒绝）rejected
    • 2：2 种事件
      1. 等待状态变化到成功状态会触发一个事件 onFulFilled
      2. 等待状态变化到失败状态又会触发另一个事件 onRejected
    • 1：1 个对象 promise 对象，promise 是 es6 原生对象

    # 面试题 - promise 的特点和状态？

    • 特点： promise 的状态不受外界影响，promise 的状态一旦发生改变，就不会再变，且任何时候能都获取到这个结果。
    • 状态： pending（进行中，等待）fulfilled（已成功）rejected（已失败）
    • 只能从 pending 变为 fulfilled 或者 从 pending 变为 rejected，一旦改变，状态就凝固了，称为 resolved。

# promise 对象的使用

  • 参考文档: https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise

  • Promise 是一个构造函数

  • 实例化 Promise 对象 通过 new 关键字实例化得到 promise 对象

  • resolve 与 reject 都是是函数，可以使用任何名字。接受任何类型的单个参数。

  • resolve (终值) 函数，可以把等待状态转换为成功状态，并携带终值

  • reject (据因) 函数，可以把等待状态转换为失败状态，并携带终值

  • then () 方法返回一个 Promise。它最多需要有两个参数：Promise 的成功和失败情况的回调函数。

  • 通过 promise 对象的 then 方法，我们可以得到终值或者据因

  • Promise 对象的 catch () 方法用于注册一个在 promise 被拒绝时调用的函数。它会立即返回一个等价的 Promise 对象，这可以允许你链式调用其他 promise 的方法。此方法是 Promise.prototype.then (undefined, onRejected) 的一种简写形式。

  • Promise.prototype.finally () finally () 方法返回一个 Promise。在 promise 结束时，无论结果是 fulfilled 或者是 rejected，都会执行指定的回调函数。这为在 Promise 是否成功完成后都需要执行的代码提供了一种方式。这避免了同样的语句需要在 then () 和 catch () 中各写一次的情况。

    # Promise 的链式调用

    • 我们可以用 Promise.prototype.then ()、Promise.prototype.catch () 和 Promise.prototype.finally () 这些方法将进一步的操作与一个变为已敲定状态的 promise 关联起来。
    • 注意: Promise 的链式调用的原因是，promise 对象调用 then (),catch (),finally () 等这些方法以后，又会返回一个 promise 对象
    • then () 方法回调函数中，可以直接返回某个值
    • 上一个 then () 方法回调函数中 return 返回的内容，会作为下一个 then () 方法回调函数的参数
    • then () 方法回调函数中，也可以返回 promise 对象
    • 建议：如果有多个 then () 链式调用的时候，catch () 方法放到最后，catch () 方法可以捕获任何一个 promise 对对象拒绝状态

# 封装 promise 方式的函数

  • 封装 promise 方式的函数，意思就是本身函数是使用回调函数的方法，我们改成 promise 方式的函数

  • 封装 promise 方式的函数核心核心核心：在该函数中要返回 promise 对象

# async 异步函数

  • async 异步函数: https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Statements/async_function

  • async 函数是使用 async 关键字声明的函数。async 函数是 AsyncFunction 构造函数的实例，并且其中允许使用 await 关键字。async 和 await 关键字让我们可以用一种更简洁的方式写出基于 Promise 的异步行为，而无需刻意地链式调用 promise。

  • async 函数的返回值为 promise 对象

  • promise 对象的结果由 async 函数执行的返回值决定

  • async 是一个关键字，用于修饰函数

  • 使用 async 修饰的函数，那么该函数会返回一个 promise 对象

  • async 函数细节：

    1. async 函数如果返回值是一个非 Promise 类型的数据，返回的值是什么，成功的终值就是什么
    2. async 函数如果返回的是一个 Promise 对象，Promise 对象返回什么状态，函数得到的就是什么状态的 promise 对象
    3. async 函数如果抛出异常，函数整体返回结果为失败状态的 Promise 对象

# await 表达式

  • await 操作符用于等待一个 Promise 兑现并获取它兑现之后的值。它只能在 "异步函数" 或者模块顶层中使用。
  • await 表达式
    1. await 关键字只能在异步函数中使用
    2. await 右侧的表达式内容，可以是 promise 对象也可以是非 promise 对象数据
  • await 表达式细节：
    • await 右侧为非 promise 对象数据，右侧是什么，await 以后得到的就是什么
    • await 右侧为 promise 对象的情况又分为两种：解决的和拒绝的
    • 如果 await 的 promise 对象状态是解决的，await 会对 promise 解决的 "终值" 做一个 "获取"
    • 如果 await 的 promise 对象状态是拒绝的，就会抛出异常，需要通过 try...catch 进行捕获处理错误信息
  • 小结：
    • await 的使用，其实说白就是，主要是对 promise 对象，它成功结果做一个获取；如果 promise 对象状态是失败的话，就需要通过 try...catch 在 catch 里边拿到失败的据因信息
    • promise 状态如果是成功，将走完所有 try 里面的代码；如果是失败则走 catch 里面的代码

# Promise 构造函数的静态方法

# ❤❤❤Promise.all()

  • Promise.all () 方法接收一个 promise 的 iterable 类型（注：Array，Map，Set 都属于 ES6 的 iterable 类型）的输入，并且只返回一个 Promise 实例，那个输入的所有 promise 的 resolve 回调的结果是一个数组。这个 Promise 的 resolve 回调执行是在所有输入的 promise 的 resolve 回调都结束，或者输入的 iterable 里没有 promise 了的时候。它的 reject 回调执行时，只要任何一个输入的 promise 的 reject 回调执行或者输入不合法的 promise 就会立即抛出错误，并且 reject 的是第一个抛出的错误信息。
  • 概念：该方法接收一个数组包含多个 promise 对象并返回一个新的 promise 对象。
  • 注意：当所有 promise 对象状态都是成功时，all 的状态也是成功。新的 promise 对象就会调用其 resolve 方法，根据数组中 promise 的顺序将对应的 resolve 返回值存入新 promise 的 resolve 数组中；如果有一个 Promise 失败 all 的状态也是失败，失败的原因是第一个失败 promise 的据因。
  • 也即是执行多个 promise 对象的操作，如果所有都是成功状态，走 then 对应的代码；如果有其中一个是拒绝状态，走 catch 对应的代码

# ❤❤❤Promise.race(iterable)

  • Promise.race (iterable) 方法返回一个 promise，一旦迭代器中的某个 promise 解决或拒绝，返回的 promise 就会解决或拒绝。

# Promise.resolve(value)

  • Promise.resolve (value) 方法返回一个以给定值解析后的 Promise 对象。如果这个值是一个 promise，那么将返回这个 promise；如果这个值是 thenable（即带有 then 方法），返回的 promise 会 “跟随” 这个 thenable 的对象，采用它的最终状态；否则返回的 promise 将以此值完成。此函数将类 promise 对象的多层嵌套展平。
  • 概念：该方法直接创建一个成功 Promise 对象

# Promise.reject()

  • Promise.reject () 方法返回一个带有拒绝原因的 Promise 对象。

# Promise.any()

  • 概念：该方法接收多个 promise 对象并返回一个 promise 对象。多个 promise 只要有一个成功执行就会被调用 promise 对象方法
  • 注意：如果有一个 promise 对象成功 any 就会返回成功，返回的值是第一个成功 promise 对象 resolve 的终值。 只有全部 promise 对象的状态是失败的时候，才会走 catch 方法对应的代码
  • 注意：any 方法还处于试验阶段，尚未被所有浏览器支持

# Promise.allSettled()

  • 概念：该方法接收多个 promise 对象并返回一个 新的 promise 对象。必须所有的 Promise 都完成（无论成功还是失败）该方法才会 resolve 一个数组，数组中包含接收所有 Promise 完成信息（status：成功 fulfilled、失败 rejected，value： 成功返回值，reason：失败原因 ）

# ES6 模块化介绍 重点 掌握

  • 介绍：在 js 中一直没有模块的概念，将 js 拆成一些互相依赖小模块。在 ES6 之前实现模块化使用的是 RequireJS 或者 seaJS（分别是基于 AMD 规范的模块化库， 和基于 CMD 规范的模块化库）。
  • ES6 引入了模块化，其设计思想是在编译时就能确定模块的依赖关系，以及输入和输出的变量。
  • 注意： ES6 的模块化自动采用严格模式。无论你是否在文件头部加上 "use strict"。ES6 的模块化分为导出（export） @与导入（import）两个模块。
  • 模块化简单理解就是指将一个大的程序文件，拆分成许多小的文件，然后将小文件组合起来。这里小的文件，就是我们的模块。
  • 模块化规范：
    • ES6 之前的模块化规范有：
    1. ❤❤❤CommonJS => NodeJS、Browserify
    2. AMD => requireJS
    3. CMD => seaJS
  • ES6 模块化语法：
    • 模块功能主要由两个命令构成：export 和 import
    • export 命令用于规定模块的对外接口
    • import 命令用于输入其他模块提供的功能

# 浏览器使用 ES6 模块化方式一

  • 在 script 标签中设置 type 为 module, 并在 script 里面导入其他模块内容

  • 通过 import 命令导入其他模块

  • ES6 引入 (导入) 模块数据语法 import

  • 概念：在 ES6 使用 export 关键字在模块中对外公开了接口之后，其他模块就可以通过 import 关键字加载对外公开接口的模块

  • 通用的导入方式：不管哪种暴露方式的数据，都可以使用的通用导入方式

    • 注意：可以把通用的导入方式理解为模块的整体加载。我们可以理解 export 关键字在公开接口时，其实是创建了一个叫做 export 的模块对象，所有接口都是该对象的属性。import 给开发者提供了 * 星号用来指定 export 模块对象。因为 js 的命名规范中 * 号不可以用做变量名，所以在模块的整体加载中需要配合 as 关键字给模块整体设置一个别名来使用。
      • 语法：
        • import * as 自定义模块名称 from "模块文件所在路径";

  • 解构赋值导入方式：

    • 语法：
      • import {名称 1, 名称 2...} from "模块文件所在路径";
      • 注意：如果变量重名会报错，可以通过 as 设置别名

  • 简便导入方式 (也叫默认导入方式)：方式只能针对默认暴露的数据才可以使用

    • 语法：
      • import 指定名称 from "默认模块 js 所在路径"

# 浏览器使用 ES6 模块化方式二

  • 先在一个 js 文件中统一导入其他模块，再使用 script 标签导入这个 js 文件

# ES6 面向对象 - Class（定义类，定义实例方法，实例属性）

  • ES6 面向对象 - Class（掌握）

  • 介绍：js 是一个脚本语言没有像强语言一样拥有类这样一个概念，在 ES6 之前创建一个实例对象都使用的是构造函数的方法。

  • ES6 引入了 class 的概念，通过 class 关键字定义类，作为对象的模板。

  • Class 可以看作一个语法糖所以他的本质就是一个 function。class 模板写法让对象原型的写法更加清晰、更像面向对象编程的语法。

  • 基本语法：

  • 类内部定义的属性 (包括方法) 都是不可枚举的

  • constructor 方法：

    • 概念： constructor 方法是类的构造函数，也是类的默认方法。通过 new 关键字创建类的实例时，会自动调用该 constructor 方法
    • 注意：
      1. 一个类必须有 constructor 方法。如果创建 class 时没有显示定义 constructor 方法，class 会默认的隐式添加一个空的 constructor 方法
      2. constructor 函数会隐式默认的返回当前实例对象 (也就是 this)。
    • 注意：开发人员可以在 constructor 函数中显式返回另一个对象作为当前 class 实例化后的对象

  • 实例属性：

    • 概念：通过 class 塑造的实例对象自身的属性 通过实例对象访问实例属性
    • ❤❤❤方法一：推荐 在构造函数 constructor 构造函数中通过 this 设置
    • 方法二： 直接写在 class 中的属性 属性名 = 属性值
    • 方法三： 使用类的 get/set 创建实例属性 难点
      • 概念：与 ES5 一样 class 可以在内部使用 get 和 set 关键字对某个属性设置 get 函数和 set 函数，拦截该属性的存取行为

# ES6 面向对象 - Class（静态属性，静态方法，私有属性）

  • 静态方法：

    • 概念：不会被实例对象所继承的方法，只能通过 class 自身调用
    • 语法：使用关键字 static 声明
    • 注意：
      1. 静态方法内部 this 指向构造函数自身，不是类的实例
      2. 静态方法可以被子类所继承

  • 静态属性：

    • 概念：不会被实例对象继承的属性就是静态属性，之前，在 class 中只有静态方法没有静态属性。后来有一个提案提出使用 static 关键字声明静态属性
    • 静态属性也是通过 class 自身访问
    • 静态属性可以被子类所继承

  • 私有属性：

    • 概念： 在之前，js 中一直不存在真正的私有属性。但是，在 ES2020 中提案 可以用 #开头声明私有属性
    • 私有属性的意思就是只能在类内部使用
    • ES2020 中提案 用 #开头声明私有属性 要记
    • 注意：私有属性只能在 constructor 方法外面声明

# ES6 面向对象 - Class（继承与 super 关键字）

  • 类的继承：

    • 概念： class 可以通过 extends 关键字实现类的继承
    • 子类通过 extends 关键字可以继承父类的实例属性，实例方法，静态属性，静态方法已经父类的私有属性
    • 如果子类中定义了 constructor () 构造函数，那么必须在 constructor () 构造函数调用 super () 超级函数
    • 注意：并且在 super () 函数调用以后，才可以使用 this 对象

  • 继承注意细节：

    • 子类可以继承父类中的静态方法，静态属性，实例方法，实例属性，子类也可以有自己独有的实例成员与静态成员
    • 一个父类允许被多个子类继承
    • 如果子类具有跟父类同名的属性和方法，则可以覆盖从父类继承过来的同名属性和方法

  • super 关键字：

    • 概念： super 就是父类构造函数，所以 super 与父类的构造函数如果需要接收同样参数，在使用类的继承时一定要记得给 super 传递参数
    • 注意：类的继承的子类中，子类如果定义了 constructor 函数 则需要在 constructor 函数中一定要调用一个 super 函数，否则会报错。
    • 注意：因为在类的继承中，子类的 this 对象必须调用父类 constructor 完成塑造，得到父类相同的属性和方法，然后进行加工返回子类的 this, 子类的 this 再加上子类自身的实例属性方法。super 其实就是父类的 constructor 方法。如果不在子类构造函数内部调用 super，子类就没有 this 对象。
    • 注意：如果子类没有显示声明 constructor 函数，ES6 会自动隐式创建 constructor 函数并在其内部调用 super 方法。如果你的子类 constructor 中不需要设置实例属性推荐隐式创建 constructor 函数
