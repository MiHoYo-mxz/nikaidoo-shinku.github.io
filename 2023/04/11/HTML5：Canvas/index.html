# Canvas 介绍

  • 参考文档: https://developer.mozilla.org/zh-CN/docs/Web/API/Canvas_API/Tutorial

  • 参考文档: https://developer.mozilla.org/zh-CN/docs/Web/API/CanvasRenderingContext2D

  • Canvas 介绍：

    • HTML5 的 canvas 元素使用 JavaScript 在网页上绘制图像。 画布是一个矩形区域，您可以控制其每一像素。 canvas 拥有多种绘制路径、矩形、圆形、字符以及添加图像的方法。

    • <canvas> 是一个可以使用脚本来绘制图形的 HTML 元素。例如，它可以用于绘制图表、制作图片构图或者制作动画。<canvas > 最早由 Apple 引入 WebKit，用于 Mac OS X 的仪表盘功能，随后被各个浏览器实现。如今，所有主流的浏览器都支持它。

    • canvas 元素

      • 概念：canvas 只可以使用 HTML 的高度和宽度属性来自定义 Canvas 的尺寸 (如果使用 css 会引起图像扭曲)，其默认大小为 300 像素 ×150 像素（宽 × 高，像素的单位是 px）。为了在 Canvas 上绘制图形，我们需要使用一个 JavaScript 上下文对象，它能动态创建图像。

    • 注意：canvas 元素除 css 可以指定的显示大小以外，canvas 画布内还有分辨率大小，css 是没办法指定分辨大小

    • canvas 标签除了可以使用通用属性以外，仅有 width、height 两个可选属性

    • 注意：<canvas> 元素需要结束标签 </canvas>。如果结束标签不存在，则文档的其余部分会被认为是替代内容，将不会显示出来。

    • canvas 画布默认宽度是 300, 高度 150 通过 canvas 的 width,height 属性可以设置画布的宽度以及高度

    • 不建议使用 css 设置宽度跟高度，因为使用 css 设置以后，会缩放画布里面的内容

    • 通过画布的 dom 对象，获取渲染上下文 (绘图工具箱)

      • HTMLCanvasElement.getContext () HTMLCanvasElement.getContext () 方法返回 canvas 的上下文，如果上下文没有定义则返回 null . 在同一个 canvas 上以相同的 contextType 多次调用此方法只会返回同一个上下文。
      • CanvasRenderingContext2D.moveTo () 将一个新的子路径的起始点移动到 (x，y) 坐标。
      • CanvasRenderingContext2D.lineTo () 使用直线连接子路径的最后的点到 x,y 坐标。
      • CanvasRenderingContext2D.stroke () 使用当前的样式描边子路径。
      • CanvasRenderingContext2D.strokeStyle 图形边线的颜色和样式。默认 #000 (黑色).
      • CanvasRenderingContext2D.beginPath () 清空子路径列表开始一个新的路径。当你想创建一个新的路径时，调用此方法。
      • 注意：如果当前路径为空，即调用 beginPath () 之后，或者 canvas 刚建的时候，第一条路径构造命令通常被视为 moveTo ()，无论实际上是什么。出于这个原因，你几乎总是要在设置路径之后专门指定你的起始位置。
      • CanvasRenderingContext2D.lineWidth 线的宽度。默认 1.0
      • CanvasRenderingContext2D.closePath () 使笔点返回到当前子路径的起始点。它尝试从当前点到起始点绘制一条直线。如果图形已经是封闭的或者只有一个点，那么此方法不会做任何操作。
      • CanvasRenderingContext2D.fillStyle 图形内部的颜色和样式。默认 #000 (黑色).
      • CanvasRenderingContext2D.fill () 使用当前的样式填充子路径。
      • CanvasRenderingContext2D.rect (x,y,width,height) 创建一个矩形路径，矩形的起点位置是 (x, y)，尺寸为 width 和 height。
      • CanvasRenderingContext2D.fillRect (x,y,width,height) 绘制填充矩形，矩形的起点在 (x, y) 位置，矩形的尺寸是 width 和 height。
      • CanvasRenderingContext2D.strokeRect (x,y,width,height) 在 canvas 中，使用当前的笔触样式，描绘一个起点在 (x, y)、宽度为 w、高度为 h 的矩形。
      • CanvasRenderingContext2D.clearRect (x,y,width,height) 设置指定矩形区域内（以 点 (x, y) 为起点，范围是 *(width, height)* ）所有像素变成透明，并擦除之前绘制的所有内容。
      • CanvasRenderingContext2D.globalAlpha 在合成到 canvas 之前，设置图形和图像透明度的值。默认 1.0 (不透明)。
      • CanvasRenderingContext2D.getLineDash () 返回当前线段样式的数组，数组包含一组数量为偶数的非负数数字。返回值 一个 Array 数组。一组描述交替绘制线段和间距（坐标空间单位）长度的数字。如果数组元素的数量是奇数，数组元素会被复制并重复。例如，设置线段为 [5, 15, 25] 将会得到以下返回值 [5, 15, 25, 5, 15, 25]。
      • CanvasRenderingContext2D.setLineDash (一个 Array 数组) 设置当前的线段样式。 参数一个 Array 数组。一组描述交替绘制线段和间距（坐标空间单位）长度的数字。如果数组元素的数量是奇数，数组的元素会被复制并重复。例如， [5, 15, 25] 会变成 [5, 15, 25, 5, 15, 25]。
      • CanvasRenderingContext2D.lineDashOffset 描述在哪里开始绘制线段。

# canvas 线条样式

  • CanvasRenderingContext2D.lineCap 线末端的类型。允许的值： butt (默认，线段末端以方形结束), round (线段末端以圆形结束), square (线段末端以方形结束，但是增加了一个宽度和线段相同，高度是线段厚度一半的矩形区域).

  • CanvasRenderingContext2D.lineJoin 定义两线相交拐点的类型。允许的值：round (通过填充一个额外的，圆心在相连部分末端的扇形，绘制拐角的形状), bevel (在相连部分的末端填充一个额外的以三角形为底的区域), miter (默认，通过延伸相连部分的外边缘，使其相交于一点，形成一个额外的菱形区域)。

# 绘制圆弧

  • 绘制圆弧 重点掌握: arc (x, y, radius, startAngle, endAngle [, anticlockwise]) 画一个以 x,y 为圆心的以 radius 为半径的圆弧（圆），从 startAngle（起始弧度） 开始到 endAngle （结束弧度）结束，按照 anticlockwise 给定的方向（默认为 false 顺时针）来生成。

  • 注意：arc () 函数中表示角的单位是弧度，不是角度。角度与弧度的 js 关系为: Math.PI 等于 180 度

# canvas 变换

  • 注意: canvas 变换是改变坐标系，改变坐标系，改变坐标系

  • CanvasRenderingContext2D.translate (x,y) 通过在网格中移动 canvas 和 canvas 原点 x 水平方向、原点 y 垂直方向，添加平移变换

  • CanvasRenderingContext2D.rotate (顺时针旋转的弧度) 在变换矩阵中增加旋转，角度变量表示一个顺时针旋转角度并且用弧度表示。

  • CanvasRenderingContext2D.scale (x,y) 根据 x 水平方向和 y 垂直方向，为 canvas 单位添加缩放变换。

# 保存图片

  • toDataURL：方法返回一个包含图片展示的 data URI 。可以使用 type 参数其类型，默认为 PNG 格式。图片的分辨率为 96dpi。

  • 参考文档: https://developer.mozilla.org/zh-CN/docs/Web/API/HTMLCanvasElement/toDataURL

  • mime 格式参考: https://www.runoob.com/http/mime-types.html

  • HTMLCanvasElement.toDataURL(type, encoderOptions )

  • HTMLCanvasElement.toDataURL () 方法返回一个包含图片展示的 data URI 。可以使用 type 参数其类型，默认为 PNG 格式。图片的分辨率为 96dpi。

  • 参数：

    • type 可选 图片格式，默认为 image/png
    • encoderOptions 可选 在指定图片格式为 image/jpeg 或 image/webp 的情况下，可以从 0 到 1 的区间内选择图片的质量。如果超出取值范围，将会使用默认值 0.92。其他参数会被忽略。

  • 返回值 包含 data URI 的 DOMString。

  • 如果画布的高度或宽度是 0，那么会返回字符串 “data:,”。

  • 如果传入的类型非 “image/png”，但是返回的值以 “data:image/png” 开头，那么该传入的类型是不支持的

  • Chrome 支持 “image/webp” 类型。

  • 注意：测试 toDataURL () 方法的时候，需要使用 http:// 或者 https:// 协议
