# Bootstrap 简介

  • Bootstrap 是由 Twitter 的 Mark Otto 和 Jacob Thornton 开发的，用于快速开发 Web 应用程序和网站的 前端框架

  • Bootstrap 是基于 HTML、CSS、JAVASCRIPT 的前端框架。

  • Bootstrap 是一款 CSS/HTML 的框架库

  • Bootstrap 集成了各种常用的前端（HTML、CSS 和 JavaScript）组件库；

  • Bootstrap 用于响应式前端布局，移动设备优先的 Web 项目开发；

  • Bootstrap 官网 https://v4.bootcss.com

  • 框架：顾名思义就是一套架构，它有一套比较完整的网页功能解决方案，而且控制权在框架本身，有预制样式库、组件和插件。使用者要按照框架所规定的某种规范进行开发。

  • 小结: Bootstrap 来自 Twitter（推特），是目前最受欢迎的前端框架。Bootstrap 是基于 HTML、CSS 和 JAVASCRIPT 的，它简洁灵活，使得 Web 开发更加快捷。

    # Bootstrap 能做什么

    • Bootstrap 是一个界面库，能让开发者快速的搭建页面
    • Bootstrap 使用移动端优先思想，因此对各种尺寸的设备支持良好
    • 在以下场景中，可能会用到 Bootstrap
    • 服务端渲染的中后台页面
    • 需要响应式布局的前、中、后台页面

    # 如何使用 Bootstrap

    • Boostrap 提供了大量的 css 样式，用于重置样式、快速布局、组件美化等

# Bootstrap 布局容器

  • .container 类 固定自适应方式

    响应式布局的容器固定宽度
    小于 576 的为超小屏幕布局容器设置宽度为 100%
    576~768 的小屏幕布局容器设置宽度为 540px
    768~992 的中等屏幕布局容器设置宽度为 720px
    992~1200 的大屏幕布局容器设置宽度为 960px
    大于 1200 的超大屏幕布局容器设置宽度为 1140px

  • .container-fluid 类 流体 100% 自适应方式 百分百宽度

  • 注意: .container 类，.container-fluid 类 会自带左右 15px 的内填充 padding

# Bootstrap 栅格系统 (重点，难点)

  • 固定容器 在不同屏幕尺寸下，会有不同的宽度

  • bootstrap 规定，栅格系统是把一行最多分成 12 列

  • 细节如下：

    1. 按照不同屏幕划分为 1~12 等份
    2. 行（.row） 可以去除父容器左右 15px 的边距
    3. 行（.row） 必须放到布局容器里面
    4. extra small：超小； sm-small：小； md-medium：中等； lg-large：大； extra large: 超大；
    5. 列（column）大于 12，多余的 “列（column）” 所在的元素将被作为一个整体另起一行排列
    6. 每一列默认有左右 15 像素的 padding
    7. 如果只写 col, col-sm , col-md, col-lg , col-xl 类，不写具体的份数，则会智能等宽划分
    8. .row 表示一行，.col-* 来表示一列
    9. 在.col - 屏幕尺寸 -* 的星号位置，还可以强制设定每列所占有的栅格列；
    10. 如果强制给每个列都设置了 具体数值分数，比如 col-3 但是没有撑满一行，那么切割以后，将不会自动填充满
    11. 可以同时为一列指定多个设备的类名，以便划分不同份数 例如 class="col-md-4 col-sm-6"
    12. 容器布局可以嵌套，但一般来说，不推荐且很少使用到
    13. 所占的栅格位正好是 12 列，超过 12 列则会换行，小于 12 列则不能 100%；
    14. 可以使用类 .w-100 意思就是设置当前元素的宽度为 100% 可以进行切割栅格栏位，实现分行操作

  • bootstrap 会把屏幕分为以下几个：

    屏幕屏幕大小类前缀
    超小屏幕<576.col-
    小屏幕>=576.col-sm-
    中屏幕>=768.col-md-
    大屏幕>=992.col-lg-
    超大屏幕>=1200.col-xl-

  • .row 表示行

  • .row 是使用 flex 弹性盒子布局
