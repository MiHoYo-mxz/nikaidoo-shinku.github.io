# 使用 JSX 书写标签语言

  • react 使用的是 jsx 语法 (JavaScriptXML) ，在原有的 js 基础上增加了 xml 语法 (严格标签语法单一型标签必须要有结束符号)。

  • 编译时需要使用 babel 转义工具将 jsx 转化为 js。

  • jsx 语法创建的 React 元素，本质上会被 babel 转译为 React.createElement ()

  • react 的本质上就是原生的 js

  • jsx 的语法，在任何 react 元素中，无论属性插值还是文本插值统一使用 "{jsx 表达式}"

    • jsx 表达式：变量的引用、函数的调用、运算符、jsx 本身

    • 声明变量 、条件控制语句 (if for while) 等等这些不属于 jsx 表达式

    • jsx 的文本插值语法是自动预防 xss 攻击

      • 如果需要将字符串以 html 元素的形式插入到 React 元素节点内部需要使用一个特殊属性 dangerouslySetInnerHTML

  • jsx 使用严格标签语法，单一型标签必须使用 / 结束符

  • jsx 变量有且仅有一个根数据 (因为一个变量只能同事存储一种数据类型)

# react 组件

  • React 中有两种组件分别是 class 组件 (类组件) 、函数组件

  • 因为现在主流是函数式编程，所以开发中流行使用函数式组件配合 hook (钩子函数)

  • React 规定无论函数组件还是类组件，组件名首字母必须大写 (现在 React 项目中默认指定组件名必须又两个及以上单词组成)

  • 类组件：

    • 类组件无法使用 hook 所以必须继承 React 提供的 Component ( 组件类 ) 拿到相关生命周期响应式方法

  • 函数组件：

    • React 的函数组件必须是使用 function 声明的函数不能使用箭头函数，React 函数组件与没有区别只是函数名的首字母必须大写在 React 脚手架中默认规定组件名至少包含两个单词

# react 脚手架

  • React 脚手架中支持空标签 Fragment, 目的包裹多个子组件但是在页面上不渲染空标签省去不必要元素嵌套
