# 使用 JSX 书写标签语言

  • react 使用的是 jsx 语法 (JavaScriptXML) ，在原有的 js 基础上增加了 xml 语法 (严格标签语法单一型标签必须要有结束符号)。

          // const test = <a href="http://www.bilibili.com">hello</a>
      // 上面的 jsx 语法创建的 React 元素,本质上会被 babel 转译为 React.createElement()
      
        const test = React.createElement('a', {
            href: 'http://www.bilibili.com',
            children: 'hello',
            className: 'demo'
        })

        const div = React.createElement('div', {
            children: test
        })

  • 编译时需要使用 babel 转义工具将 jsx 转化为 js。

  • jsx 语法创建的 React 元素，本质上会被 babel 转译为 React.createElement ()

  • react 的本质上就是原生的 js

  • jsx 的语法，在任何 react 元素中，无论属性插值还是文本插值统一使用 "{jsx 表达式}"

    • jsx 表达式：变量的引用、函数的调用、运算符、jsx 本身

    • 声明变量 、条件控制语句 (if for while) 等等这些不属于 jsx 表达式

            let num = 7;

      let h2 = <h2>hello</h2>;

      const demo = (
        <div>
          <h2>{num}</h2>
          <p className={num >= 18 ? "active" : ""}>
            {num >= 18 ? "成年" : "未成年"}
          </p>
          <div>{h2}</div>
        </div>
      );

    • jsx 的文本插值语法是自动预防 xss 攻击

      • 如果需要将字符串以 html 元素的形式插入到 React 元素节点内部需要使用一个特殊属性 dangerouslySetInnerHTML

              let str = "<h2>hello</h2>";

      const element = <div>{str}</div>; // 渲染时 str 变量的值将会以文本节点字符串的形式插入到页面中

      // 如果需要将字符串以html元素的形式插入到React元素节点内部需要使用一个特殊属性
      const element1 = <div dangerouslySetInnerHTML={{ __html: str }}></div>;

  • jsx 使用严格标签语法，单一型标签必须使用 / 结束符

          const div = <div />;
      const input = <input />;

  • jsx 变量有且仅有一个根数据 (因为一个变量只能同事存储一种数据类型)

# react 的 diff 算法

  • vue 的 diff 算法与 react 的 diff 算法区别：

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <script src="./react.development.js"></script>
    <script src="./react-dom.development.js"></script>
    <script src="./babel.min.js"></script>
    <script src="./vue@3.3.4.js"></script>
  </head>

  <body>
    <div id="app"></div>
    <!-- 下面script type表示内部的代码使用上面在线转译工具转化为js代码 -->
    <script type="text/babel">
      const app = document.getElementById("app");
      const root = ReactDOM.createRoot(app);

      function render() {
        let date = new Date().toLocaleTimeString();

        let el = (
          <div>
            <h2>React时钟</h2>
            <p>当前事件 {date}</p>
          </div>
        );

        root.render(el);
      }

      render();

      setInterval(render, 1000);
    </script>

    <div id="demo">
      <h2>Vue时钟</h2>
      <p>当前事件 {{ date }}</p>
    </div>

    <script>
      const { createApp } = Vue;

      createApp({
        data() {
          return {
            date: new Date().toLocaleTimeString(),
          };
        },
        mounted() {
          setInterval(() => {
            this.date = new Date().toLocaleTimeString();
          }, 1000);
        },
      }).mount("#demo");
    </script>

    <!-- 对比页面中 React 与 Vue 时间发生改变时,更新的节点不同可以发现, Vue 更新整个p元素节点,而react只更新了 { date } 差值的文本节点

原因: React 元素都是 immutable 不可变的。当元素被创建之后，你是无法改变其内容或属性的。一个元素就好像是动画里的一帧，它代表应用界面在某一时间点的样子。

React 虚拟DOM 通过新旧 React 元素对象的替换告诉 ReactDOM 更新页面(响应式), 虚拟DOM是新的直接替换掉旧的,但是如果真实dom也直接使用替换是比较影响性能的,真实DOM更新使用了react Diff算法

ReactDOM 会比较新旧DOM对象类型是否不同,若不同卸载当前DOM创建新的DOM, 若相同再比较内部的属性或子节点,在更新时只会更新改变的部分(递归比较)

所以 react diff算法可以总结成一句话, react的 diff算法只会更新必要的部分 -->
  </body>
</html>

# react 组件

  • React 中有两种组件分别是 class 组件 (类组件) 、函数组件

  • 因为现在主流是函数式编程，所以开发中流行使用函数式组件配合 hook (钩子函数)

  • React 规定无论函数组件还是类组件，组件名首字母必须大写 (现在 React 项目中默认指定组件名必须又两个及以上单词组成)

  • 类组件：

    • 类组件无法使用 hook 所以必须继承 React 提供的 Component ( 组件类 ) 拿到相关生命周期响应式方法

            class MyComponent extends React.Component {
        constructor(props) {
          super(props);

          // 类组件的数据存放在 state 的属性中
          this.state = {
            name: "老王",
            age: 12,
          };
        }

        addAge() {
          console.log(this, "aaa");
          // React 修改state因为不像vue给属性绑了get set
          // 他修改state需要用到 this.setState

          // this.state.age++ 无效

          this.setState({
            age: this.state.age + 1, // React 属性不可变性,请使用替换的方式修改数据
          });
        }

        // react 的类组件有一个必须设置的生命周期 render 表示组件渲染时执行的函数, 该函数必须要有返回值,返回的值就是当前组件渲染在页面上的样子
        // 返回值 必须是React Node类型(能渲染在页面上的数据类型) 不能是对象及函数, 在React中普通对象不是 React child
        render() {
          let { name, age } = this.state;

          return (
            <div>
              <h3>username: {name}</h3>
              <p> age: {age} </p>
              {/* 注意react事件绑定会导致React组件内的方法中this指向错误, 下面的代码调用后this为undefined */}
              <button onClick={this.addAge}>add age1</button>
              {/* 解决方法使用箭头函数 */}
              <button onClick={() => this.addAge()}>add age2</button>
            </div>
          );
        }
      }

  • 函数组件：

    • React 的函数组件必须是使用 function 声明的函数不能使用箭头函数，React 函数组件与没有区别只是函数名的首字母必须大写在 React 脚手架中默认规定组件名至少包含两个单词

      <!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <script src="./react.development.js"></script>
    <script src="./react-dom.development.js"></script>
    <script src="./babel.min.js"></script>
</head>

<body>
    <div id="app"></div>
    <!-- 下面script type表示内部的代码使用上面在线转译工具转化为js代码 -->
    <script type="text/babel">

        const app = document.getElementById('app')
        const root = ReactDOM.createRoot(app)

        const { useState } = React


        // React的函数组件必须是使用function声明的函数不能使用箭头函数, React函数组件与没有区别只是函数名的首字母必须大写在React脚手架中默认规定组件名至少包含两个单词

        function MyComponent() {
            // 函数组件的返回值必须是 React Node
            // 在早期 react 函数组件当时被称为无状态组件,意思是函数组件无法创建存放数据的状态,早期函数组件只负责一些简单UI封装的组件
            // 后续React版本的更新中提出Hook的概念,使得函数式编程一下火了起来, 函数式组件配合react Hook 可以在函数组件中创建state与声明了
            // 使用React 提供 useState hook创建函数组件的状态,因为React的状态都是不可变的,useState 返回一个数组,数组第一项是state的值,而第二项是修改该state的方法
            const [name, setName] = useState('老王')
            const [age, setAge] = useState(19)

            function test() {
                console.log(213);
                
                setAge(age + 1)
                
                // setAge(age + 1) // 这里仍然是旧的age
                // 注意: 上面的代码理论上来说应该是 age+2 , 但是真实效果是只加了一, 原因是 setAge 可能是异步的

                // setState 支持两种格式格式一: setAge(新值) 格式二 setAge((newAge) => newAge + 1)  这里的newAge\就是上次更新后最新的age值,返回值就是再次更新替换的新值

                setAge(newAge => newAge + 1)
            
            }

            return (
                <div onClick={test}>
                    <p>name : {name} </p>
                    <p>age : {age}</p>
                </div>
            )
        }


        root.render(<MyComponent />)

        // 使用官方脚手架创建 react 项目

        // npx create-react-app my-app --template typescript

    </script>
</body>

</html>
