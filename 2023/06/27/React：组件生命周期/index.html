# 类组件的生命周期

  • 类组件的常用声明周期只有 5 个，分别是：

    • constructor 实例化
    • render 渲染
    • componentDidMount 页面第一次挂载完毕
    • componentDidUpdate 组件数据发生更新后页面重新渲染完毕
    • componentWillUnmount 组件将要被卸载，用于清除副作用

  • ❗注意：在使用 React 脚手架进行开发时，我们会发现打开页面后生命周期会执行 componentDidMount -> componentWillUnmount -> componentDidMount

    • 真实情况理论上只会触发 componentDidMount
    • 原因是在脚手架开发中默认认为你是开发者模式，开发者模式中 react 会故意在页面还没挂载完毕时 触发一次 componentDidMount 和一次 componentWillUnmount
    • 因为开发人员有时偷懒有时是疏忽 在 componentDidMount 开启的副作用并没有在 componentWillUnmount 中清除，这样就容易造成一些 bug 变成产品仍然存在
    • react 为了避免这个问题，故意在每个组件第一次挂载前自动多触发一套 componentDidMount -> componentWillUnmount 生命周期行为目的为了在开发调试中暴露更多的 bug

<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <script src="../day01/react.development.js"></script>
    <script src="../day01/react-dom.development.js"></script>
    <script src="../day01/babel.min.js"></script>
</head>

<body>
    <div id="app"></div>
    <script type="text/babel">
        // 状态提升: 多个组件公用相同的状态(state),开发人员就需要将 state 提升到多个组件的公共父组件中并通过 props 传参给子组件使用
        const { useState } = React
        const app = document.getElementById('app')
        const root = ReactDOM.createRoot(app)

        // 类组件的常用声明周期只有5个分别是
        // constructor 实例化
        // render 渲染
        // componentDidMount 页面第一次挂载完毕
        // componentDidUpdate 组件数据发生更新后页面重新渲染完毕
        // componentWillUnmount 组件将要被卸载, 用于清除副作用

        class ComponentA extends React.Component {
            constructor(props) {
                super(props)
                this.state = {
                    name: '老王',
                    age: 18
                }
            }

            componentDidMount() {
                console.log('页面第一次被渲染,这里适合发起网路请求开启计时器操作DOM等有副作用的操作');
            }

            componentDidUpdate(prevProps, prevState) {
                console.log('数据发生更新,页面重新渲染完成后执行');
                console.log('改变前的State', prevState)
                console.log('改变后新的state', this.state);
                console.log('该声明周期中一般用于实现 watch或计算属性相关功能 功能');
            }

            componentWillUnmount() {
                console.log('组件被卸载时触发的声明周期,该方法中要清除componentDidMount中创建的有副作用的操作');
            }

            render() {
                const { name, age } = this.state

                return (
                    <div>
                        <p>
                            姓名: {name} <br />
                            <input type="text" value={name} onChange={e => this.setState({ name: e.target.value })} />
                        </p>
                        <p>
                            年龄 {age}
                            <button onClick={() => this.setState({age: age + 1})}>add btn</button>
                        </p>
                    </div>
                )
            }
        }

        // 注意在使用React 脚手架进行开发时,我们会发现打开页面后生命周期会执行 componentDidMount -> componentWillUnmount -> componentDidMount
        // 真实情况理论上只会触发 componentDidMount
        // 原因是在脚手架开发中默认认为你是开发者模式, 开发者模式中react 会故意在页面还没挂载完毕时 触发一次 componentDidMount 和一次 componentWillUnmount
        // 因为开发人员有时偷懒有时是疏忽 在 componentDidMount 开启的副作用并没有在 componentWillUnmount 中清除, 这样就容易造成一些bug变成产品仍然存在
        // react为了避免这个问题,故意在每个组件第一次挂载前自动多触发一套 componentDidMount -> componentWillUnmount 生命周期行为目的为了在开发调试中暴露更多的bug

        root.render(<ComponentA />)

    </script>
</body>

</html>

# 函数组件的生命周期

  • 在函数组件中生命周期只有一个 hook 叫做 useEffect, 这个 useEffect 等价于 类组件中的 componentDidMount componentWillUpdate componentDidUpdate componentWillUnmount。

  • 语法：useEffect (setup, dependencies?)

    • setup 生命周期钩子函数
    • dependencies 数组内部存放指定 state 名，只有在指定的 state 发生改变时才会触发该 Effect 的 componentWillUpdate \ componentDidUpdate 从而就可以实现类似 vue 的 watch 计算属性效果

  • 若 dependencies 为空数组时，当前的 Effect 就会成为一个只会在页面挂载和卸载时触发的钩子函数

    <!DOCTYPE html>
<html lang="en">

  • react 中所有 hook 其底层原理都是闭包，这就使得在使用 hook 过程中会产生一些问题，以下是一个经典案例：

          function ComponentA() {
        const [age, setAge] = useState(5);

        useEffect(() => {
          let timer = setInterval(() => {
            // 这里访问age发现值一只是初始值 5
            console.log(age);
            // 原因是所有的hook其底层原理都是闭包, useEffect(setup, dependencies?) setup函数就会被闭包缓存起来

            // 如果依赖是一个空数组,就表示 useEffect 只在页面挂载时时触发过一次  setup 该时间被创建后对 age的引用是当时外部age state的初始值并被闭包缓存,后续若 setup 没有重新执行其内部age永远是5
            // 问如何在 dependencies 为空数组的情况下拿到新的state, 解决方法通过setState的回调形式
            setAge(age + 1);
          }, 1000);

          return () => {
            clearInterval(timer);
          };
        }, []);

        return (
          <div>
            <p>
              姓名: {name} <br />
              <input
                type="text"
                value={name}
                onChange={(e) => setName(e.target.value)}
              />
            </p>
            <p>
              年龄 {age}
              <button onClick={() => setAge(age + 1)}>add btn</button>
            </p>
          </div>
        );
      }
    • 如果需要 dependencies 为 [] 时，任然能通过定时器改变 state，可以像下面案例一样将 setAge (age + 1) 写成 setAge ((age) => age + 1) 形式。
    • 除了下面说到的闭包原理外，还存在另外一个原因造成 age 一直获取的都是初始值：
      • react 的所有 hook 会生成一个对象，对象中会保存当前组件的所有 state，结合闭包的话就会造成 setAge (age + 1) 中的 age 指向的是 useEffect 所生成对象中的 age 而不是外部 const [age, setAge] = useState (5) 中的 age，只有 hook 中 setup 执行才会更新对象中的 state，但 dependencies 为 [] 的话 setup 只会在组件挂载时执行，也就是说只执行了一次，对象中的 state 永远都是初始值，但如果写成 setAge ((age) => age + 1) 形式，箭头函数中参数指向的是 const [age, setAge] = useState (5) 中的 age，每次都会更新为新的值。
          function ComponentA() {
        const [age, setAge] = useState(5);

        useEffect(() => {
          let timer = setInterval(() => {
            // 这里访问age发现值一只是初始值 5

            // 原因是所有的hook其底层原理都是闭包, useEffect(setup, dependencies?) setup函数就会被闭包缓存起来

            // 如果依赖是一个空数组,就表示 useEffect 只在页面挂载时时触发过一次  setup 该时间被创建后对 age的引用是当时外部age state的初始值并被闭包缓存,后续若 setup 没有重新执行其内部age永远是5

          }, 1000);

          return () => {
            clearInterval(timer);
          };
        }, []);
