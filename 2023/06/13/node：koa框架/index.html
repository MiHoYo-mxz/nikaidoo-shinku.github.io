# koa 框架

  • 基于 node.js 平台的下一代 web 开发框架

  • 上下文：

    • koa 将 node 服务器的 request 和 response 封装到了 context 对象中，作为中间件函数的参数。该封装为编写 web 应用提供了很多 API, 简化在 HTTP 服务器开发中频繁用到的接口。

    • 常见 API：

      1. cxt.req 等价于 node request 对象，koa 还提供了 cxt.request API 他其实就是 cxt.req 别名

      2. cxt.res 等价于 node response 对象，koa 还提供了 cxt.response API 他其实就是 cxt.res 别名

      3. cxt.app koa 的服务器实例对象

      4. cxt.cookie.get (name, [option]) 获取 cookie

      5. cxt.cookie.set (name, value, [option]) 通过 Option 设置 cookie

        • maxAge: 一个数字，表示从 Date.now () 得到的毫秒数.
        • expires: 一个 Date 对象，表示 cookie 的到期日期 (默认情况下在会话结束时过期).
        • path: 一个字符串，表示 cookie 的路径 (默认是 /).
        • domain: 一个字符串，指示 cookie 的域 (无默认值).
        • secure: 一个布尔值，表示 cookie 是否仅通过 HTTPS 发送 (HTTP 下默认为 false, HTTPS 下默认为 true). 阅读有关此参数的更多信息.
        • httpOnly: 一个布尔值，表示 cookie 是否仅通过 HTTP (S) 发送，, 且不提供给客户端 JavaScript (默认为 true).
        • sameSite: 一个布尔值或字符串，表示该 cookie 是否为 "相同站点" cookie (默认为 false). 可以设置为'strict', 'lax', 'none', 或 true (映射为'strict').
        • signed: 一个布尔值，表示是否要对 cookie 进行签名 (默认为 false). 如果为 true, 则还会发送另一个后缀为 .sig 的同名 cookie, 使用一个 27-byte url-safe base64 SHA1 值来表示针对第一个 Keygrip 键的 cookie-name=cookie-value 的哈希值。此签名密钥用于检测下次接收 cookie 时的篡改.
        • overwrite: 一个布尔值，表示是否覆盖以前设置的同名的 cookie (默认是 false). 如果是 true, 在同一个请求中设置相同名称的所有 Cookie（无论路径或域）是否在设置此 Cookie 时从 Set-Cookie 消息头中过滤掉.

      6. cxt.throw ([status],[msg],[properties]) 抛出一个网络状态码异常，默认值为 500

        • 注意：name required 是用户级错误并不是错误的具体内容，表示消息只适用于客户端响应。通常情况下开发人员并不会向客户端泄漏具体故障详细信息。但是你可以根据需要将 properties 对象传递到错误中，通过该方式向客户端发送 **** 人机友好型错误 ****，便于让用户了解错误报告。

  • Request 别名 (简写 API)：

    • ctx.header
    • ctx.headers
    • ctx.method
    • ctx.url
    • ctx.href
    • ctx.path
    • ctx.query
    • ctx.querystring
    • ctx.host
    • ctx.socket
    • ctx.protocol

  • Response 别名：

    • ctx.body=
    • ctx.status

  • koa 的中间件：

    • koa-router koa 路由中间件
    • koa-body 解析 post 请求的 body 请求体，支持表单文件上传
    • koa-bodyparser 解析 post 请求的 body 请求体，不支持表单文件上传可以配合 @koa/multer 模块解析表单文件上传
    • koa-session 基于 cookie 登录验证
    • koa-jwt token 令牌登录验证
    • koa-view 配合 ejs 服务端渲染
    • koa-ejs koa 的 ejs 语法渲染末班
    • koa-static 管理服务器静态文件目录中间件
    • koa-compress 压缩 koa 中间件
