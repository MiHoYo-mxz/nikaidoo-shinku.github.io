# Node.js Stream (流)

  • Stream 是一个抽象接口，Node 中有很多对象实现了这个接口。例如，对 http 服务器发起请求的 request 对象就是一个 Stream，还有 stdout（标准输出）。

  • Stream 流是一个 node.js 提供的模块，目的支持 "流数据" 这种数据解构类似生活中讲话一样有序的依次输入或输出，流是 node 中核心概念之一它是一种有效读写方法，网络通信或任何端到端的交互方法。

    使用 stream 时，不用将文件从头读到尾存入内存再对其进行处理，stream 是逐块读取并处理，无需将其保存到内存中

  • Node.js，Stream 有四种流类型：

    • Readable - 可读操作 - 读取流 (fs.createReadStream ())。
    • Writable - 可写操作 - 写入流 (fs.createWriteStream ())。
    • Duplex - 可读可写操作 - 读写流 (net.Socket).
    • Transform - 操作被写入数据，然后读出结果 - 类似于读写流但是输入是输出变形 (for example, zlib.createDeflate ())。

  • 流数据类型对象都是 EventEmitter, 它触发三个事件 data 表示流数据可以读取，end 流事件结束 error 流事件失败。

  • 所有的 Stream 对象都是 EventEmitter 的实例。常用的事件有：

    • data - 当有数据可读时触发。

    • end - 没有更多的数据可读时触发。

    • error - 在接收和写入过程中发生错误时触发。

    • finish - 所有数据已被写入到底层系统时触发。

  • 读取流中读取数据时是分块一部分一部分进行读取

  • 为什么要使用流：

    1. 无需在内存中加载大量数据等待加载完成再输入输出，从而节省内存效率

    2. 流处理数据所需事件更少，他可以在流开始时立即处理数据，而不是等待整个数据处理完成再使用，从而提高时间效率

  • pipe 方法：

    • 读取流 (输入流) 都有一个 pipe() 方法，他就是建立一个管道，将读取流的流内容通过管道流入到其他流中
